# Theme

The theming system provides a typed, component-scoped styling layer built on top of `tss-react` and MUI. Components define their styles with `createStyles`, receive the active theme at render time, and re-render automatically when the theme changes via `ThemeProvider`.

## Overview

Each component calls `createStyles(stylesOrDelegate)` at module level to get a `useStyles` hook. Inside the component, calling `useStyles()` returns `{ css, theme, tools, device }`. CSS class names are generated by `tss-react` and scoped automatically. Themes are plain objects conforming to the `Theme` shape defined in `themes/ThemeModel.ts`; they are injected through `ThemeProvider` and consumed by `useTheme`. The `mergeThemes` utility deep-merges a partial override onto a base theme, making it easy to build variant themes.

## Key exports

| Export | Description |
|--------|-------------|
| `createStyles` | Module-level factory — accepts a static style map or a `(theme, tools) => styles` delegate and returns a `useStyles` hook that resolves class names at render time. |
| `ThemeProvider` | Wraps children in `ThemeContext.Provider`, mounts MUI's theme, applies global CSS from `theme.text`, and renders any `theme.fonts` as `@font-face` rules. |
| `useTheme` | Returns the current `ThemeContextProps` (including the active `theme` object and an `isValid` flag) from the nearest `ThemeProvider`. |
| `useDevice` | Reactively reports the current `DeviceType` (`'web' \| 'tablet' \| 'mobile'`) from `matchMedia`; re-renders on rotation / resize across the breakpoint. Also surfaced as `useStyles().device`. |
| `mergeThemes` | Deep-merges a `DeepPartial<Theme>` override onto a base `Theme`, returning a new merged theme object. |
| `createAnimationKeyFrame` | Converts a CSS keyframe definition object into a `tss-react` `keyframes` string for use in styles. |
| `colors` | Utility object with `lighten(color, pct)` and `darken(color, pct)` helpers backed by the `color` library. |
| `DefaultTheme` | The default `Theme` implementation — use directly or pass to `mergeThemes` as a base. |

## Usage

```tsx
import { createStyles, ThemeProvider, DefaultTheme, mergeThemes } from '@anupheaus/react-ui';

// 1. Define component styles once at module level
const useStyles = createStyles((theme, tools) => ({
  root: {
    backgroundColor: theme.fields.content.normal.backgroundColor,
    borderRadius: theme.fields.content.normal.borderRadius,
    ...tools.applyTransition('background-color'),
  },
}));

// 2. Consume styles inside the component
function MyComponent() {
  const { css } = useStyles();
  return <div className={css.root}>Hello</div>;
}

// 3. Provide a theme at the root (optionally merged with overrides)
const customTheme = mergeThemes(DefaultTheme, {
  text: { family: 'Inter, sans-serif' },
});

function App() {
  return (
    <ThemeProvider theme={customTheme}>
      <MyComponent />
    </ThemeProvider>
  );
}
```

## Device-responsive styling

Components style for touch devices via the theme's `pseudoClasses.tablet` / `pseudoClasses.mobile` keys, which in production are real `@media(pointer: coarse)` queries (the breakpoints live in `useDevice/deviceQueries.ts`, shared with `useDevice` so JS and CSS never drift):

```tsx
const useStyles = createStyles(({ pseudoClasses }) => ({
  input: {
    padding: '0 8px',
    [pseudoClasses.tablet]: { padding: '8px 12px' }, // larger touch target on touch devices
  },
}));
```

For logic (not just styling) read the device directly: `const { device } = useStyles();` or `const device = useDevice();`.

### Simulating a device in Storybook

Real `@media(pointer: coarse)` queries never match on a desktop browser, so Storybook can't trigger them naturally. Use the **Device** toolbar (Web / Tablet / Mobile) to render any story as a given device, or fix a story to one via `parameters: { device: 'tablet' }` (or the `device` shortcut on `createStorybookComponentStates`). Under the hood `StorybookDeviceProvider` swaps the device pseudo-classes for `.is-storybook-tablet` / `.is-storybook-mobile` ancestor-class selectors and reports the chosen `device` on the theme context. These `.is-storybook-*` selectors exist **only** in the Storybook build — the shipped `DefaultTheme` is never modified.
